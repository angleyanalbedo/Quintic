# 竞品差距分析 (Gap Analysis)

本文档旨在记录 Quintic 与行业成熟凸轮设计软件（如 Siemens TIA Portal CamTool, CODESYS SoftMotion, Beckhoff TwinCAT Cam Design）之间的功能与体验差距，以便指导后续开发方向。

## 1. 核心功能对比 (Core Features)

| 功能维度 | Quintic (当前状态) | Siemens TIA CamTool | CODESYS SoftMotion | 差距评级 |
| :--- | :--- | :--- | :--- | :--- |
| **数学内核** | 基础多项式 (Poly5), 简单的三角函数 | 极其丰富 (Poly5/7/8, B-样条, 自动平滑) | 丰富 (Poly5/7, 样条, 线性) | ⭐⭐⭐ (高) |
| **连续性控制** | 自动前向推导 (V/A 自动传递) | **自动求解** (C2/C3 连续性自动约束求解器) | 自动/手动混合 | ⭐⭐ (中) |
| **坐标系支持** | 相对/绝对, 时间/角度 | 极其灵活 (主/从轴缩放, 偏移, 模态切换) | 灵活 | ⭐⭐ (中) |
| **多段拼接** | 支持, 但缺乏平滑过渡处理 | **自动圆角/过渡 (Blending)** | 支持过渡段插入 | ⭐⭐⭐ (高) |
| **物理仿真** | 仅运动学 (V/A/J) | **动力学** (扭矩, 惯量, 摩擦力, 电机选型) | 动力学集成 | ⭐⭐⭐ (高) |

## 2. 用户体验与交互 (UI/UX)

| 交互维度 | Quintic | 商业软件 (Siemens/CODESYS) | 改进方向 |
| :--- | :--- | :--- | :--- |
| **可视化编辑** | 混合模式 (表格 + 拖拽控制点) | **全交互式**: 拖拽曲线手柄, 拖拽关键点直接修改参数 | 增强交互体验 |
| **实时反馈** | 毫秒级重算 (优秀) | 实时, 但大型项目可能卡顿 | 保持当前性能优势 |
| **错误提示** | 物理极限报警 (V/A/J 红色高亮) | **智能诊断**: 红色高亮冲突段, 物理极限报警, 连续性断裂警告 | 增加动力学报警 |
| **辅助工具** | 无 | 测量尺, 缩放锁定, 多层叠加对比 (Trace) | 增加图表工具栏 |

## 3. 生态集成 (Integration)

*   **代码生成**:
    *   **Quintic**: 仅 CSV 导出 (离散数据点)。
    *   **Siemens/CODESYS**: 直接生成 PLC 代码块 (DB/Function Block/Struct), 或直接编译进项目。
    *   **差距**: **巨大**。工业现场工程师更喜欢直接生成的代码（如 ST 数组或结构体），而非导入 CSV 文件，因为 CSV 需要额外的解析代码。

*   **在线调试 (Online)**:
    *   **Quintic**: 纯离线设计工具。
    *   **Siemens/CODESYS**: 在线监控, 支持将**实际编码器反馈曲线**与**理论设计曲线**叠加对比 (Trace 功能)，用于诊断机械磨损或 PID 滞后。
    *   **差距**: 需要开发 OPC UA / Modbus 通讯接口 (Roadmap 6.2)。

## 4. 追赶计划 (Action Plan)

根据上述差距，建议的开发优先级排序如下：

1.  **离散逻辑轨道 (Discrete Logic)**: 实现运动与 IO 的协同 (Cam Switch)，这是包装机械的核心需求。
2.  **PLC 代码导出 (Code Gen)**: 打通最后一公里，支持 IEC 61131-3 ST 代码或 Siemens DB 导出，替代 CSV。
3.  **动力学分析 (Kinematics)**: 从“运动学”升级到“动力学”，提供 RMS 扭矩和功率预测，对标高端分析工具。
4.  **样条插值 (Splines)**: 引入 B-Spline 以支持更复杂的平滑曲线拟合。
